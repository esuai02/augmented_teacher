# 1) 📚 체계적 진도형 – World Expert Intelligence 설계

## W: 세계관(Worldview) 정렬

* 핵심 신념: “진도는 전략, 보정은 일상.”
* 목표: 교과-단원 선형 진도 + 주간 진단-보정 루프.
* 교사 모토(원문 연결): *학습 한계 상황을 적시에 해소 · 매주 진단 후 계획 재조정*
* 기본 전개: 단원 마스터리 → 누적 복습(7:3) → 월간 커리 리셋.
* 보조 세계관 연결 규칙

  * ✏️ 성과집중형(시험모드) : 시험 D‑30 진입 시 우선순위 1순위로 스위칭
  * 🎯 개인맞춤형 : 기초 결손 감지 시 하위 스택으로 즉시 덧대기
  * ⚡ 목표달성형(미션) : 동기저하/루틴 붕괴 시 단기 스트릭로 회복
  * 🧠 사고력중심형 : 이해도 딥다이브가 필요할 때 블록 단위 삽입
  * 🚀 자율학습형 : 상위권·자율성 높을 때 주차 설계권 일부 위임
  * 🔍 인지적 도제형 : 풀이사고 시연/스캐폴딩 필요 시 세션 내 삽입

## X: 문맥지능(Context)

* 필수 컨텍스트 수집(주 1회 업데이트)

  * 학년/범위, 최근 단원 스코어, 오답 패턴, 선행/복습 비율, 시험 캘린더, 학습시간 로그, 결손 개념 리스트.
* 스위칭 트리거

  * D‑30: ✏️시험모드 온(기출 3회독·파이널 리뷰)
  * 진도이탈 >10% or 달성률 <80%: 🎯맞춤·⚡미션 병행
  * 사고질문 증가·정답률 불안정: 🔍도제/🧠사고력 블록 주입
  * 자율실행률 ≥70%·보고 품질 우수: 🚀자율학습 블렌딩

## S: 구조지능(Structure)

* 표준 변수(편집 유지용)

  * `주차`, `단원목록`, `목표숙련(%, A/B/C)`, `선행:복습=7:3`, `주간테스트`, `월간리셋`, `이탈임계치(10%)`, `D-day`.
* KPI

  * 주간 진도달성 ≥90%, 단원 마스터리 ≥80%, 오답감소율 주차당 ≥20%, 스터디타임(집중) ≥12h/주.
* 데이터 모델(예시 JSON)

```json
{
  "mode":"curriculum",

    {"week":2,"units":["미분기초","벡터연산"],"targetMastery":0.85}
  ],
  "thresholds":{"slip":0.1,"examSwitch":30},
  "kpi":{"pace":0.9,"mastery":0.8,"errDown":0.2,"focusHours":12}
}
```

## P: 절차지능(Procedure)

* 4주 스프린트 루프

  1. 진단·로드맵(주0) → 2) 주간 스프린트(주1\~3) → 3) 통합리뷰·리셋(주4).
* 수업 루틴 (Every Session)

  * Before(5’): 전차시 오답 체크/오늘 목표 명시
  * During(40’): 핵심개념→유형 스텝→미니퀴즈
  * After(15’): 백지복습·오답 라벨링·숙제 큐
* 시험 캘린더 절차

  * D‑30 계획 정렬 → D‑14 실전세트 주2회 → D‑7 파이널리뷰 → D‑2 컨디션 루틴.

## E: 실행지능(Execution)

* 교사 체크리스트

  * [ ] 주차 진도표 업데이트  \[ ] 단원별 마스터리 채점
  * [ ] 이탈 학생 알림·보정안 전달  \[ ] 시험모드 스위치 점검
* 학생 루틴(📚 curriculum 학생모드 부연)

  * 매일 정시 학습, 주간 진도 체크, 7:3 유지, 월간 점검, 번아웃시 페이스 다운.
* 자동화 매핑(‘mathking/management’ 활용)

  * 진도율 분석·로드맵 자동생성, 이탈 탐지·리마인더, 출제빈도/예상문제 트래킹, 사고과정 기록.

### 실행용 에이전트 프롬프트 세트

1. **RoadmapAgent (주간 로드맵 생성)**

```
역할: 📚 체계적 진도형 로드맵 설계자
입력: 단원목록, 시험일(D-day), 최근 오답, 목표숙련도, 선행:복습 비율(7:3)
규칙: ① 주당 2~3단원 ② 복습블록 30% ③ D-30부터 ✏️시험모드 섞기
출력: 주간 학습표(일자/단원/문항/복습큐), KPI, 위험·보정안
```

2. **PaceMonitor (이탈 감시 & 보정)**

```
입력: 계획 대비 실제 진도, 오답율, 집중시간
트리거: 이탈 >10% 또는 달성률 <80%
처방: 🎯맞춤 모듈(기초 개념 2h/일) + ⚡미션(5개/일, 10/5 포모도로)
```

3. **ExamSwitch (시험모드 전환)**

```
트리거: D-30
처방: 기출 3회독 플랜, 백지복습 매일, 모의실전 주 2회, 파이널리뷰 리스트
```

4. **CognitiveCoach (사고력 삽입)**

```
상황: 풀이 이유 설명 불능/전이 실패
구성: 모델링→코칭→스캐폴딩→명료화→탐색 순서로 20분 블록
```

5. **SelfLed Orchestrator (자율 블렌딩)**

```
조건: 자율실행률 ≥70%
권한위임: 주간 계획 초안 학생 작성→교사 검수, 월 2회 피드백
```

## R: 성찰지능(Reflection)

* 주간 리플렉션 6문

  1. 가장 많은 시간을 잡아먹은 개념/유형은? 왜?
  2. “계획→실행” 갭 %와 원인 1가지
  3. 가장 효과적이었던 복습 방식 1개
  4. 전이된 문제/전이 실패 문제 각 1개
  5. 다음 주 제거할 활동 1개·증가할 활동 1개
  6. 시험모드 스위치 필요성(Y/N)·근거
* 개선 규칙

  * 갭 >20% → 범위 축소, 미션 난이도 재조정
  * 전이 실패 지속 → 🔍도제 블록 2배 증량
  * 번아웃 신호 → 페이스 80%로 1주 감속

## T: 트래픽 지능(Traffic & Transfer)

* 정보흐름 설계

  * 월간(커리 리셋), 주간(로드맵·진단), 일간(미션·숙제), 세션(오답피드백).
* J-커브 준비

  * D‑30 시험 트래픽 폭증 대비: 파이널 큐·오답은행 정렬, 리마인더 자동화.
* 단절지점 탐지

  * ‘무지 노트’ 공란 지속·보고 누락 2회 → 알림+개입 세션.

## A: 추상화/시간 지능(Aftermath)

* 분기 회고: 단원 전이 지도(개념→유형→통합문항) 재정의, 템플릿/오답태깅 규칙 표준화.
* 재사용 자산: 파이널리뷰 리스트, 기출 빈도 매트릭스, 학생별 UX 로그(🎯개인맞춤형에 전송).
