# 9) 🔭 호기심 해결 중심형 – World Expert Intelligence 설계

## W: 세계관 (Worldview) 정렬

* 핵심 신념: **“배움의 출발점은 질문, 그 자체다.”**
* 목표: 수학의 단원·개념·문제 현상 속에서 \*\*왜?\*\*라는 질문을 중심에 두고 탐구를 확장.
* 철학:

  * 점수·성적·시험과 분리된 순수 학습.
  * 호기심이 향하는 곳으로 학습 경로를 유연하게 조정.
* 운영 원리:

  * GPTs와 맞춤형 생성 콘텐츠를 활용해 즉시 탐구 지원.
  * 전체 커리큘럼 지도와 ‘탐구 컴퍼스’를 통해 위치·방향성 제안.
* 보조 세계관 연결:

  * 📚 체계적 진도형: 호기심 탐구를 커리큘럼 맥락에 맞춰 구조화
  * 🎯 개인맞춤형: 호기심 주제를 학생 성향·수준에 맞춤
  * 🧠 사고력 중심형: 깊이 있는 사고 훈련과 결합
  * 🚀 자율학습형: 탐구 주제·경로를 전적으로 위임
  * 🔍 인지적 도제형: 탐구 과정 모델링·스캐폴딩
  * 🕒 시간 피드백형: 호기심 탐구 시간을 밀도 있게 관리

---

## X: 문맥지능 (Context)

* 필수 컨텍스트 수집:

  * 현재 호기심 주제, 탐구 동기, 단원 내 위치, 커리큘럼 지도상 진행도, 탐구 질문 목록, 선호 학습 도구(GPTs 활용 방식 포함).
* 스위칭 트리거:

  * 호기심 주제 탐구 정체 → 새로운 질문·연계 개념 제시
  * 탐구 범위 지나치게 확산 → 📚로 구조화
  * 탐구 깊이 부족 → 🧠/🔍 심화
  * 자율성 부족 → 🚀 보조
  * 시간 관리 실패 → 🕒 결합

---

## S: 구조지능 (Structure)

* 표준 변수:

  * `탐구주제`, `탐구질문수`, `커리큘럼위치`, `다음방향제안`, `GPT활용기록`.
* KPI:

  * 주간 탐구 질문 ≥ 3개 생성
  * 탐구 완료 주제 ≥ 1개/주
  * 커리큘럼 지도 기반 방향 결정률 ≥ 80%
  * GPT 응답 활용도 ≥ 90%
* 데이터 모델 예시:

```json
{
  "mode":"curiosity",
  "currentTopic":"피보나치 수열",
  "questions":["왜 황금비율과 연결될까?","자연에서 발견 예시?"],
  "curriculumPosition":"수열 단원 중급",
  "nextDirection":"수학사 탐구",
  "gptUsageCount":5
}
```

---

## P: 절차지능 (Procedure)

* 1일 루틴:

  1. **질문 생성**: 오늘 탐구할 ‘왜?’ 1\~3개
  2. **탐구 확장**: GPTs·자료·시뮬레이션 활용
  3. **개념 지도 확인**: 현재 위치·다음 방향 제안
  4. **정리 기록**: 탐구 결과·새 질문·연계 개념 저장
* 주간 루프:

  * 월\~목: 호기심 탐구 세션
  * 금: 탐구 정리·발표(간단 보고서)
  * 주말: 탐구 포트폴리오 업데이트

---

## E: 실행지능 (Execution)

* 교사 체크리스트:

  * [ ] 학생 질문 생성·기록 확인
  * [ ] 탐구 방향·범위 검토
  * [ ] 커리큘럼 지도·컴퍼스 제안 업데이트
  * [ ] GPTs 활용 질·양 점검
* 학생 루틴:

  * 오늘의 질문 작성
  * GPTs·자료로 탐구
  * 개념 지도에서 위치·다음 단계 확인
  * 탐구 노트 업데이트
* 자동화 매핑:

  * **mathking**: 탐구 개념 시각화·문제 연계
  * **management**: 탐구 주제·방향 기록, 질문 생성 리마인드

## 실행용 에이전트 프롬프트 세트

1. **CuriosityCompass**

```
역할: 🔭 호기심 탐구 경로 설계자
입력: 현재 주제·질문·커리큘럼 위치
규칙: ① 관련 개념·연계 분야 제시 ② 난이도·깊이 단계별 추천
출력: 탐구 경로·다음 질문 후보·참고 자료
```

2. **QuestionExpander**

```
입력: 주제·현재 질문
출력: 새로운 탐구 질문 3~5개, 연계 단원
```

3. **PortfolioUpdater**

```
입력: 주간 탐구 기록
출력: 탐구 포트폴리오(질문·결과·연계 개념)
```

## R: 성찰지능 (Reflection)

* 주간 리플렉션 질문:

  1. 이번 주 가장 재미있었던 질문·이유
  2. 예상 외로 흥미로웠던 발견
  3. 탐구 중 어려웠던 점
  4. 다음 주 더 깊이 파고들 주제
* 개선 규칙:

  * 질문 생성률 ↓ → QuestionExpander로 주제 자극
  * 방향 결정률 ↓ → 📚·🚀 모드 결합
  * 깊이 부족 → 🧠/🔍 병행

---

## T: 트래픽 지능 (Traffic & Transfer)

* 정보흐름:

  * 질문·탐구 데이터 → 📚·🧠·🔍 모드로 환류
  * 커리큘럼 지도 기반 경로 추천 → 🚀 자율학습 모드 결합
* J-커브:

  * 호기심 주제의 확산과 연결성이 높을 때 지식 네트워크 급성장
* 단절지점:

  * 질문 생성 중단·기록 누락 → 즉시 호기심 자극 세션

---

## A: 추상화 지능 (Aftermath)

* 분기별: 호기심 질문·탐구 결과 아카이브
* 축적 자산: 주제별 탐구 지도, 질문 유형 DB, GPTs 응답·활용 패턴 분석

---

## 상황별 블렌딩 레시피

* **호기심 확산 제어 필요** → 🔭 + 📚
* **깊이 있는 사고 필요** → 🔭 + 🧠/🔍
* **자율 탐구 강화** → 🔭 + 🚀
* **시간 관리 필요** → 🔭 + 🕒
* **약점 기반 탐구** → 🔭 + 🎯
