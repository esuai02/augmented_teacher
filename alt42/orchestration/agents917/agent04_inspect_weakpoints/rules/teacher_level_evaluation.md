## 🎯 영역별 상세 평가

### 1️⃣ 페르소나 기반 분석 (18/20점, A-)

#### ✅ 강점
- **7가지 활동별 페르소나 분석 룰**: 개념이해, 유형학습, 문제풀이(60개), 오답노트, 질의응답, 복습, 포모도르
  - R9~R13: 각 활동에서 페르소나 분석 및 DB 업데이트
  - 문제풀이 60개 페르소나 매칭 시스템 구축
- **페르소나 기반 맞춤형 피드백**: 페르소나 매칭 결과에 따른 컨텐츠 배치 전략
- **페르소나 리셋 및 재분석**: 정답률 급변 시 페르소나 재분석 룰 (R2b, R2b2)

#### ⚠️ 부족한 점
- **수학 학습 스타일 분류 부재**: 
  - 선생님: "계산형 학생 vs 개념형 학생 vs 응용형 학생"
  - 현재: MBTI 기반 페르소나만 있고, 수학 특화 학습 스타일 분류 없음
- **문제풀이 단계별 페르소나 세분화 부족**:
  - 선생님: "문제 해석 단계에서 막히는 학생 vs 풀이 과정에서 실수하는 학생"
  - 현재: 60개 페르소나 언급만 있고, 단계별 세부 분석 로직 부재

**점수: 18/20점** (90%)

### 2️⃣ 학습 활동 선택 전략 (12/20점, C+)

#### ✅ 강점
- **7개 활동 카테고리 구분**: 개념이해, 유형학습, 문제풀이, 오답노트, 질의응답, 복습, 포모도르
- **시나리오별 활동 선택 전략**: S1~S7 시나리오에 따른 활동 선택 로직
- **정답률 기반 활동 전환**: 정답률 급변 시 활동 전환 전략 (R2)

#### ❌ 현저히 부족한 점

**수학 교과 특화 활동 선택 전략 부재:**

1. **단원별 활동 우선순위 미반영**
   - 선생님: "함수 단원은 개념→유형→문제풀이 순서가 필수"
   - 현재: 일반적 순서만 있고, 단원별 특성 반영 없음

2. **문제 유형별 활동 선택 전략 부재**
   - 선생님: "기본형은 유형학습 중심, 심화형은 문제풀이 중심"
   - 현재: 활동 선택 시 문제 유형 고려 없음

3. **교재별 활동 전략 부재**
   - 선생님: "쎈은 유형학습 중심, 개념원리는 개념이해 중심"
   - 현재: 교재 정보 수집은 있으나 활동 선택에 반영 안 됨

4. **단원 연계성 기반 활동 선택 부재**
   - 선생님: "함수와 그래프는 함께 공부하면 효과적"
   - 현재: 단원 간 연계성 고려 없음

**점수: 12/20점** (60%)

---

### 3️⃣ 수학 교과 특화 지식 (8/20점, D)

#### ❌ 현저히 부족

**수학 특화 요소 누락:**

1. **수학 단원별 난이도 DB 부재** (-5점)
   - 선생님: "함수는 어렵고(난이도 8/10), 통계는 상대적으로 쉬워(난이도 4/10)"
   - 현재: 단원별 난이도 정보 없음, 학생 입력에만 의존

2. **문제 유형 분류 DB 부재** (-4점)
   - 선생님: "함수 단원 유형: 일차함수 그래프, 함수식 구하기, 함수의 활용"
   - 현재: 기본/유형/심화만 있고, 단원별 상세 유형 분류 없음

3. **단원 간 연계성 DB 부재** (-3점)
   - 선생님: "방정식을 먼저 해야 함수를 제대로 할 수 있어"
   - 현재: 단원 선후관계 미반영

4. **교재별 특성 DB 부재** (-2점)
   - 선생님: "쎈은 유형 많고, 개념원리는 설명 좋고"
   - 현재: 교재명만 수집, 교재별 특성 및 활용 전략 없음

5. **출제 빈도/비중 DB 부재** (-2점)
   - 선생님: "함수 단원은 시험에서 30% 출제"
   - 현재: 출제 비중 정보 없음

**점수: 8/20점** (40%)

### 4️⃣ 난이도 조정 전략 (11/15점, C+)

#### ✅ 강점
- **정답률 기반 난이도 조정**: 40~70% 성장구간 유지 전략 (R8)
- **급변 대응**: 정답률 급락/급상승 시 즉각 조정 (R2, R2a)
- **워밍업 전략**: 난이도 과다 시 개념 리프레시 + 쉬운 문제 (R1a, R8a)

#### ⚠️ 부족한 점

1. **단원별 난이도 기준 부재** (-2점)
   - 선생님: "함수 단원 기본형 = 다른 단원 유형형"
   - 현재: 모든 단원 동일 기준 적용

2. **계산 실수 vs 개념 오류 구분 미흡** (-2점)
   - 선생님: "계산 실수는 반복 연습, 개념 오류는 개념 재학습"
   - 현재: R4에서 오류 원인 분류는 있으나, 수학 특화 대응 전략 부재

3. **학생 수준별 난이도 기준 부재** (-1점)
   - 선생님: "하위권은 정답률 50%가 적정, 상위권은 70%가 적정"
   - 현재: 모든 학생 동일 기준 (40~70%)

**점수: 11/15점** (73%)

### 5️⃣ 학생 수준별 차별화 (7/10점, C)

#### ✅ 강점
- **페르소나 기반 차별화**: 각 활동별 페르소나 분석 및 맞춤형 피드백
- **감정 상태 고려**: 피로/저항감에 따른 차별화 전략 (R7)

#### ⚠️ 부족한 점

1. **수학 학습 스타일 분류 부재** (-2점)
   - 선생님: "계산형 학생은 반복 연습, 개념형 학생은 개념 정리 강조"
   - 현재: MBTI만 있고, 수학 특화 학습 스타일 없음

2. **수준별 활동 비중 차별화 부족** (-1점)
   - 하위권: 개념 60% + 유형 30% + 문제풀이 10%
   - 중위권: 개념 30% + 유형 50% + 문제풀이 20%
   - 상위권: 개념 10% + 유형 30% + 문제풀이 60%
   - 현재: 수준별 비중 조정 전략 없음

**점수: 7/10점** (70%)

## 🔧 개선 우선순위 제안

### 🔴 긴급 (현직 수준 도달 필수)

1. **수학 단원별 난이도/학습시간 DB 구축**
   - 학년별, 단원별 평균 난이도 (1~10 척도)
   - 단원별 평균 학습 시간 (개념/유형/심화)
   - 출제 비중 데이터
   - **데이터 수집 방법**: 선생님 체크리스트 입력
     - "함수 단원: 난이도 8/10, 개념 3시간, 유형 4시간, 심화 2시간, 출제비중 30%"
   - **예상 룰 추가**: R15_unit_difficulty_based_activity_selection

2. **문제 유형 분류 DB 구축**
   - 단원별 상세 문제 유형 분류
   - 유형별 목표 정답률 기준
   - **데이터 수집 방법**: 선생님 텍스트 입력
     - "함수 단원 유형: 일차함수 그래프(기본형, 목표 정답률 80%), 함수식 구하기(유형형, 목표 정답률 70%), 함수의 활용(심화형, 목표 정답률 60%)"
   - **예상 룰 추가**: R16_problem_type_based_selection

3. **단원 간 연계성 DB 구축 및 활용**
   - 학년별 단원 선후관계 정의
   - 연계 단원 동시 학습 전략
   - **데이터 수집 방법**: `math_unit_relations.yaml` 참조 (agent02에서 구축됨)
   - **예상 룰 추가**: R17_unit_relation_based_learning_order

4. **계산 실수 vs 개념 오류 차별화 전략**
   - R4 보완: 수학 특화 대응 전략 추가
   - 계산 실수 → 반복 연습 모듈
   - 개념 오류 → 개념 재학습 모듈
   - **데이터 수집 방법**: 학생 설문
     - "이 문제를 틀린 이유는? (A) 계산 실수 (B) 개념 이해 부족 (C) 문제 해석 오류"
   - **예상 룰 추가**: R4d_error_type_specific_intervention

### 🟡 중요 (수준 향상 필요)

5. **수학 학습 스타일 분류 시스템**
   - 계산형/개념형/응용형 분류
   - 학습 스타일별 활동 선택 전략
   - **데이터 수집 방법**: 학생 설문 (agent01 온보딩에서 수집 가능)

   - **데이터 수집 방법**: 선생님 텍스트 입력
     - "쎈: 유형 많음, 유형학습 중심 추천. 개념원리: 설명 좋음, 개념이해 중심 추천"
   - **예상 룰 추가**: R19_textbook_based_activity_strategy

   - **데이터 수집 방법**: 선생님 체크리스트 입력

   - **데이터 수집 방법**: 선생님 텍스트 입력
   - **예상 룰 수정**: R8 (단원별 기준 반영)
