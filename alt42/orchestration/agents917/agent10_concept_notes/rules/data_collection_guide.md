# Agent 10 - 데이터 수집 방안 가이드

> **작성일**: 2025-01-27  
> **목적**: 현재 데이터가 부족한 경우 학생 설문, 선생님 체크리스트, 텍스트 입력을 통한 데이터 수집

---

## 개요

Agent 10의 룰 개선을 위해 필요한 데이터 중 현재 시스템에 없는 데이터는 다음과 같은 방법으로 수집합니다:

1. **선생님 체크리스트**: 구조화된 항목에 체크/입력
2. **학생 설문**: 학생이 직접 응답
3. **선생님 텍스트 입력**: 자유 형식 텍스트 입력

---

## 1. 단원별 기준값 수집

### 1.1 목적
학생의 필기량/시간을 비교할 기준값 설정

### 1.2 수집 방법

#### 방법 A: 선생님 체크리스트
```
📋 단원별 기준값 설정 체크리스트

□ 함수 단원
  - 평균 필기량: [___] 회
  - 평균 학습시간: [___] 시간
  - 난이도: [쉬움/보통/어려움] 선택

□ 방정식 단원
  - 평균 필기량: [___] 회
  - 평균 학습시간: [___] 시간
  - 난이도: [쉬움/보통/어려움] 선택

□ 통계 단원
  - 평균 필기량: [___] 회
  - 평균 학습시간: [___] 시간
  - 난이도: [쉬움/보통/어려움] 선택

... (각 단원별로 반복)
```

#### 방법 B: 선생님 텍스트 입력
```
💬 단원별 기준값을 입력해주세요

예시:
- 함수 단원: 평균 필기량 200회, 평균 학습시간 3시간, 난이도 어려움
- 방정식 단원: 평균 필기량 150회, 평균 학습시간 2시간, 난이도 보통
- 통계 단원: 평균 필기량 100회, 평균 학습시간 1.5시간, 난이도 쉬움
```

### 1.3 데이터 저장 형식
```yaml
unit_baselines:
  - unit_id: "middle1_function"
    unit_name: "함수"
    avg_stroke_count: 200
    avg_study_time_hours: 3
    difficulty: 8
  - unit_id: "middle1_equation"
    unit_name: "일차방정식"
    avg_stroke_count: 150
    avg_study_time_hours: 2
    difficulty: 6
```

## 2. 학습 단계별 기준값 수집

### 2.1 목적
각 학습 단계의 이상적 필기량/시간 기준 설정

### 2.2 수집 방법

#### 선생님 체크리스트
```
📋 학습 단계별 기준값 설정 체크리스트

□ 개념요약 단계
  - 평균 필기량: [___] 회
  - 평균 시간: [___] 분
  - 이상적 패턴: [간단 필기/자세한 필기] 선택

□ 개념이해 단계
  - 평균 필기량: [___] 회
  - 평균 시간: [___] 분
  - 이상적 패턴: [간단 필기/자세한 필기] 선택

□ 개념체크 단계
  - 평균 필기량: [___] 회
  - 평균 시간: [___] 분
  - 이상적 패턴: [간단 필기/자세한 필기] 선택

□ 예제퀴즈 단계
  - 평균 필기량: [___] 회
  - 평균 시간: [___] 분
  - 이상적 패턴: [간단 필기/자세한 필기] 선택
```

### 2.3 데이터 저장 형식
```yaml
stage_baselines:
  - stage_id: "concept_summary"
    stage_name: "개념요약"
    avg_stroke_count: 30
    avg_time_minutes: 10
    ideal_pattern: "간단 필기"
  - stage_id: "concept_understanding"
    stage_name: "개념이해"
    avg_stroke_count: 100
    avg_time_minutes: 20
    ideal_pattern: "자세한 필기"
```

## 3. 오답 유형 분류 수집

### 3.1 목적
오답 발생 시 원인 분류 (계산 실수/개념 오류/적용 오류)

### 3.2 수집 방법

#### 방법 A: 선생님 체크리스트
```
📋 오답 유형 분류 체크리스트

문제: [문제 내용 표시]
학생 답안: [학생 답안 표시]
정답: [정답 표시]

이 오답의 원인은?
○ 계산 실수
○ 개념 오류
○ 적용 오류
○ 기타: [___]
```

#### 방법 B: 학생 설문
```
📝 오답 원인 자가 평가 설문

문제: [문제 내용 표시]

이 문제를 틀렸을 때 이유는 무엇인가요?
(A) 계산 실수 (예: 부호, 계산 실수)
(B) 이해 못함 (예: 개념을 모름)
(C) 적용 못함 (예: 개념은 아는데 문제에 적용 못함)
(D) 기타: [___]
```

### 3.3 데이터 저장 형식
```yaml
wrong_answers:
  - problem_id: "p001"
    student_answer: "3"
    correct_answer: "-3"
    error_type: "computational"  # computational/conceptual/application
    classified_by: "teacher"  # teacher/student/auto
    classification_date: "2025-01-27"
```

## 4. 필기 패턴 데이터 수집

### 4.1 목적
필기 간격, 지우기 횟수, 필기 순서 등 패턴 분석

### 4.2 수집 방법

#### 방법 A: 자동 수집 (우선)
- 화이트보드 시스템에서 자동으로 수집
- stroke_timestamps: 필기 시간 데이터
- eraser_events: 지우기 이벤트 데이터
- stroke_positions: 필기 위치 데이터

#### 방법 B: 학생 설문 (자동 수집 불가 시)
```
📝 필기 패턴 자가 평가 설문

이 개념을 공부할 때:
1. 필기를 많이 했나요?
   (A) 많이 했어요 (B) 보통이에요 (C) 적게 했어요

2. 필기 중간에 막힌 적이 있나요?
   (A) 자주 막혔어요 (B) 가끔 막혔어요 (C) 막힌 적 없어요

3. 필기를 여러 번 지웠나요?
   (A) 많이 지웠어요 (B) 조금 지웠어요 (C) 거의 안 지웠어요

4. 필기 순서가 정리되어 있었나요?
   (A) 잘 정리되어 있었어요 (B) 보통이에요 (C) 산만했어요
```

## 5. 개인화 기준 설정 수집

### 5.1 목적
학생별 맞춤 기준값 설정 (일반 기준값과 다를 수 있음)

### 5.2 수집 방법

#### 선생님 텍스트 입력
```
💬 학생별 맞춤 기준값 설정

학생: [학생 이름/ID]

이 학생의 특별한 기준값:
- 함수 단원: 필기량 150회면 충분 (일반 기준 200회보다 낮음)
- 학습 속도: 평균보다 느림 (시간은 1.2배 소요)
- 학습 스타일: 시각형 (필기 강조 피드백 효과적)

특별 고려사항:
- 집중력이 약해서 짧은 시간 학습 선호
- 자신감이 낮아서 격려 중심 피드백 필요
```

### 5.3 데이터 저장 형식
```yaml
student_custom_baselines:
  - student_id: "s001"
    custom_unit_baselines:
      - unit_id: "middle1_function"
        custom_stroke_count: 150
        custom_study_time_hours: 3.6
    learning_characteristics:
      - "집중력 약함"
      - "자신감 낮음"
    feedback_preferences:
      - "격려 중심"
      - "짧은 피드백"
```

## 6. 피드백 템플릿 커스터마이징

### 6.1 목적
선생님 경험 기반 효과적인 피드백 템플릿 수집

### 6.2 수집 방법

#### 선생님 텍스트 입력
```
💬 효과적인 피드백 템플릿 공유

학생 수준: [하위권/중위권/상위권]
상황: [필기량 부족/오답 발생/재방문 패턴 등]

이런 학생에게는 이런 피드백이 효과적:
"○○○를 잘했어요! 다음에는 ○○○를 더해보면 더 좋을 것 같아요."

예시:
- 하위권 + 필기량 부족: "오늘 필기를 열심히 했어요! 조금만 더 정리하면 완벽해요."
- 중위권 + 오답 발생: "개념은 이해했는데 문제 적용이 어려웠네요. 유형 연습을 해보면 좋을 것 같아요."
- 상위권 + 재방문: "이미 잘 알고 있는 개념인데 다시 확인하고 있네요. 새로운 도전을 해보면 어떨까요?"
```

## 7. TTS 활용 데이터 수집

### 7.1 목적
TTS 재생 횟수/타이밍 분석을 통한 청각 학습 의존도 파악

### 7.2 수집 방법

#### 방법 A: 자동 수집 (우선)
- TTS 재생 이벤트 로그
- 재생 횟수, 재생 타이밍 (필기 전/중/후)

#### 방법 B: 학생 설문 (자동 수집 불가 시)
```
📝 TTS 활용 패턴 설문

이 개념을 공부할 때:
1. TTS(음성 설명)를 들었나요?
   (A) 많이 들었어요 (B) 조금 들었어요 (C) 안 들었어요

2. TTS를 언제 들었나요?
   (A) 필기 전에 들었어요 (B) 필기 중에 들었어요 (C) 필기 후에 들었어요

3. TTS를 몇 번 들었나요?
   (A) 여러 번 들었어요 (B) 한두 번 들었어요 (C) 안 들었어요
```

## 8. 페이지별 체류 시간 데이터 수집

### 8.1 목적
페이지별 체류 시간 분석을 통한 막힘 구간 감지

### 8.2 수집 방법

#### 방법 A: 자동 수집 (우선)
- 페이지 방문/이탈 이벤트 로그
- 페이지별 체류 시간 계산

#### 방법 B: 학생 설문 (자동 수집 불가 시)
```
📝 학습 패턴 설문

이 개념을 공부할 때:
1. 어떤 페이지에서 시간을 많이 보냈나요?
   [페이지 제목/번호 선택]

2. 그 페이지에서 막힌 적이 있나요?
   (A) 많이 막혔어요 (B) 조금 막혔어요 (C) 막힌 적 없어요

3. 왜 그 페이지에서 시간을 많이 보냈나요?
   (A) 이해하기 어려웠어요 (B) 자세히 공부하고 싶었어요 (C) 다른 이유
```

## 10. 데이터 수집 우선순위

### 우선순위 1 (필수)
1. 단원별 기준값 수집
2. 오답 유형 분류 수집
3. 학습 단계별 기준값 수집

### 우선순위 2 (중요)
4. 필기 패턴 데이터 자동 수집
5. TTS 활용 데이터 자동 수집

## 11. 데이터 검증 및 품질 관리

### 11.1 데이터 검증 규칙
- 필수 필드 누락 확인
- 값의 범위 검증 (예: 필기량은 양수)
- 일관성 검증 (예: 어려운 단원은 필기량 많아야 함)

### 11.2 데이터 품질 관리
- 선생님 입력 데이터: 수동 검토
- 학생 설문 데이터: 자동 검증 + 이상치 필터링
- 자동 수집 데이터: 로그 확인 + 에러 처리
