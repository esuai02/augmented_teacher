# 용어집

버전: v0.1 (초안) / 작성일: 2025-10-26 / 담당: BTS팀

## 용도와 범위
- 목적: 온보딩 입력(설문, 학습스타일, 신규상담 녹취)을 표준 용어로 정규화하여 에이전트 간 상호운용성과 추론 일관성을 보장.
- 결과물: 각 용어의 정의, 키(key), 타입/값 도메인, 정규화 규칙, 온톨로지 매핑, PII 등급, 출처/신뢰도 기본 정책.

## 공통 메타데이터 규칙
- PII 등급: none | low | medium | high (전화·주소 등은 high)
- 출처(sourceType): intake_survey | learning_style_form | counseling_transcript | teacher_input | system_inferred
- 신뢰도 기본값 r_source: 0.85 | 0.9 | 0.6 | 0.95 | 0.7 (의사결정_지식 문서 기준)
- 시간 메타: collectedAt(수집시각), validFrom/To(유효기간), updatedBy(작성자/역할)
- 근거(Evidence): { sourceType, timestamp, reliabilityScore, uri? }

## 1) 학생·제출자 기본
- 학생 ID
  - 정의: 시스템 내부 학생 식별자
  - 키: studentId
  - 타입/도메인: integer(양의 정수)
  - 정규화: 숫자 외 문자 제거
  - 온톨로지 매핑: Student.studentId
  - PII: low
  - 출처/신뢰도: intake_survey | r=0.85

- 학생 이름
  - 정의: 학생의 표시명(이모지 포함 가능)
  - 키: studentDisplayName, studentLegalName?
  - 타입/도메인: string
  - 정규화: displayName은 원형 보존, legalName은 이모지·특수문자 제거/정규화
  - 온톨로지 매핑: Student.name
  - PII: high
  - 출처/신뢰도: intake_survey | r=0.85

- 제출자 역할
  - 정의: 설문 작성자 역할(예: teacher)
  - 키: formSubmitterRole
  - 타입/도메인: enum{student, guardian, teacher, admin}
  - 정규화: 소문자 스네이크 케이스
  - 온톨로지 매핑: Evidence.annotator.role
  - PII: low
  - 출처/신뢰도: intake_survey | r=0.85

## 2) 연락처·주소
- 학생 연락처
  - 정의: 학생 휴대전화 번호
  - 키: studentPhone
  - 타입/도메인: phone(E.164/KR), string
  - 정규화: 하이픈 삭제→E.164(+82) 표준, 마스킹 저장(* 보존 정책 적용)
  - 온톨로지 매핑: Student.contact.phone
  - PII: high
  - 출처/신뢰도: intake_survey | r=0.85

- 부모님 연락처
  - 정의: 보호자 연락처(부/모)
  - 키: guardianContacts[{relation, phone}]
  - 타입/도메인: array of {enum{father,mother,other}, phone}
  - 정규화: 중복 제거, 1인 정책 위반 시 충돌 플래그
  - 온톨로지 매핑: Guardian.contact.phone; (Student)-[HAS_GUARDIAN]->(Guardian)
  - PII: high
  - 출처/신뢰도: intake_survey | r=0.85

- 주소
  - 정의: 학생 거주지 텍스트
  - 키: addressFreeform, addressCode?
  - 타입/도메인: string; 행정코드(optional)
  - 정규화: 우편번호/법정동 매핑 시 addressCode에 저장
  - 온톨로지 매핑: Student.address
  - PII: high
  - 출처/신뢰도: intake_survey | r=0.85

## 3) 학교/과정/학년

## 정합성·충돌 규칙(요약)
- guardianContacts는 정책상 1인 입력 권장이나, 다중 입력 시 충돌 플래그 후 우선순위(법정 보호자) 규칙 적용
- 개념공부/심화 진도 중복·역행 입력 시 최신성(d_recency)과 출처(r_source)로 우선 결정, 병기 가능
- 이름 필드에 이모지 포함 시 displayName으로만 보존, 법적 문서엔 legalName 사용
