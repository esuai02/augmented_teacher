## 반응 코드 개요

| 코드 | 반응 유형 | 학생 UX 관점 |
|------|----------|-------------|
| A | 수용형 (Acceptance) | 개입을 긍정적으로 받아들이는 상황 |
| R | 저항형 (Resistance) | 개입에 저항하거나 거부하는 상황 |
| N | 무응답형 (No Response) | 개입에 반응하지 않는 상황 |
| D | 지연반응형 (Delayed) | 개입에 늦게 반응하는 상황 |

### 반응 하위 유형

| 주 유형 | 하위 유형 | 설명 | 대응 전략 |
|---------|-----------|------|-----------|
| A | reinforcement | 강화가 필요한 초기 수용 | 긍정 강화, 지속 모니터링 |
| A | consolidation | 안정화 단계의 수용 | 일관성 유지, 자율성 확대 |
| A | enthusiastic | 적극적이고 열정적인 수용 | 도전 기회 제공, 과부하 방지 |
| R | explicit | 명시적인 거부 표현 | 자율성 존중, 선택권 부여 |
| R | passive | 겉으로는 동의하나 실질적 저항 | 행동 모니터링, 원인 탐색 |
| R | negotiation | 조건부 협상 시도 | 협상 가능 영역 제시 |
| N | silent | 완전한 무응답 | 다양한 채널 시도, 부모 협조 |
| N | minimal | 최소한의 형식적 반응 | 흥미 유발, 구체적 질문 |
| D | processing | 숙고를 위한 의도적 지연 | 시간 존중, 적절한 리마인더 |
| D | avoidance | 회피를 위한 지연 | 시작 문턱 낮추기, 동행 제안 |

## A: 수용형 반응 컨텍스트

### 상황 설명
학생이 개입 제안을 긍정적으로 받아들이는 상황. 가장 이상적인 반응이지만, 지속성과 내면화 여부 확인이 중요.

### 학생이 경험하는 세부 UX 컨텍스트

#### CTX-A-01: 즉각적 수용 상황
- **트리거**: 개입 제안 후 즉시 긍정적 반응
- **학생 경험**: "네, 알겠습니다"라고 바로 응답
- **관찰 지표**:
  - 응답 시간: 30초 이내
  - 응답 내용: 긍정적 동의
  - 추가 질문: 실행 방법 관련
- **시스템 대응**: 이해도 확인, 구체적 실행 계획 제공
- **주의점**: 단순 순응인지 진정한 수용인지 확인 필요

#### CTX-A-02: 질문 후 수용 상황
- **트리거**: 이유/방법 질문 후 납득하여 수용
- **학생 경험**: "왜 그렇게 해야 해요?" → 설명 듣고 → "알겠어요, 해볼게요"
- **관찰 지표**:
  - 질문 유형: 이해 확인, 방법론
  - 수용 후 반응: 구체적 계획 수립
- **시스템 대응**: 충분한 설명 제공, 학생 참여 유도
- **성공 지표**: 설명 후 자발적 동의

#### CTX-A-03: 열정적 수용 상황
- **트리거**: 개입을 적극적으로 환영
- **학생 경험**: "좋아요! 더 해도 돼요?"
- **관찰 지표**:
  - 응답 톤: 긍정적, 열정적
  - 추가 요청: 더 많은 과제/활동 요구
- **시스템 대응**: 과부하 방지, 단계적 목표 설정
- **주의점**: 번아웃 위험 모니터링

#### CTX-A-04: 관계 기반 수용 상황
- **트리거**: 교사/시스템에 대한 신뢰로 수용
- **학생 경험**: "선생님이 시키시니까요", "믿을게요"
- **관찰 지표**:
  - 관계 언급: 교사/시스템 신뢰 표현
  - 의존적 경향: 독립적 판단 부족
- **시스템 대응**: 내재적 동기 연결, 자기 결정감 부여
- **주의점**: 관계 의존성 → 자율성으로 전환 필요

#### CTX-A-05: 조건부 수용 상황
- **트리거**: 특정 조건 하에서 수용
- **학생 경험**: "이거 하면 [보상]이 있어요?", "대신 [조건]은요?"
- **관찰 지표**:
  - 협상 시도: 조건 제시
  - 교환적 사고: 이익 계산
- **시스템 대응**: 명확한 기대치 설정, 내재적 가치 전달
- **주의점**: 조건 의존성 점진적 감소 필요

#### CTX-A-06: 회복형 수용 상황
- **트리거**: 이전 저항/무응답에서 전환
- **학생 경험**: "이번에는 해볼게요...", "다시 시작해볼까요?"
- **관찰 지표**:
  - 이전 상태: R 또는 N에서 전환
  - 조심스러운 태도: 확인 질문 많음
- **시스템 대응**: 작은 성공 적극 인정, 안전한 환경 유지
- **성공 지표**: 회귀 없이 안정적 수용 유지

## R: 저항형 반응 컨텍스트

### 상황 설명
학생이 개입 제안에 저항하거나 거부하는 상황. 저항의 유형과 강도 파악이 중요.

### 학생이 경험하는 세부 UX 컨텍스트

#### CTX-R-01: 명시적 거부 상황
- **트리거**: 개입 제안에 직접적인 거부
- **학생 경험**: "싫어요", "안 할래요", "안 해요"
- **관찰 지표**:
  - 거부 표현: 명확하고 단호함
  - 감정 상태: 불만, 분노, 반항
- **시스템 대응**: 강요 금지, 선택권 부여, 이유 탐색
- **주의점**: 저항 강화 방지, 관계 유지

#### CTX-R-02: 논리적 반박 상황
- **트리거**: 개입의 논리/타당성에 대한 이의 제기
- **학생 경험**: "그게 왜 필요한데요?", "이건 맞지 않는 것 같아요"
- **관찰 지표**:
  - 질문 유형: 근거, 대안 요구
  - 태도: 비판적이나 건설적
- **시스템 대응**: 논리적 설명 제공, 토론 환영, 학생 의견 존중
- **성공 가능성**: 논리적 설득 시 수용 전환 높음

#### CTX-R-03: 소극적 저항 상황
- **트리거**: 겉으로는 동의하나 실질적으로 미이행
- **학생 경험**: "네..." (하지만 행동 안 함), "깜빡했어요"
- **관찰 지표**:
  - 언행 불일치: 말과 행동 다름
  - 반복적 미이행: 같은 패턴 반복
- **시스템 대응**: 행동 모니터링 강화, 작은 단위 과제, 원인 탐색
- **주의점**: N(무응답)으로 전환 방지

#### CTX-R-04: 감정적 반발 상황
- **트리거**: 개입이 정서적 상처나 트라우마를 건드림
- **학생 경험**: "왜 저한테만 그래요!", "너무해요!"
- **관찰 지표**:
  - 감정적 반응: 화남, 울음, 격앙
  - 비합리적 거부: 논리보다 감정 우선
- **시스템 대응**: 감정 먼저 수용, 진정 시간 제공, 정서적 지지
- **위험 신호**: 자해 언급, 극단적 회피 시 즉시 에스컬레이션

#### CTX-R-05: 협상형 저항 상황
- **트리거**: 완전 거부가 아닌 조건 협상 시도
- **학생 경험**: "절반만 하면 안 돼요?", "이것 대신 저거 하면요?"
- **관찰 지표**:
  - 대안 제시: 분량/방법/시간 조정 요청
  - 타협 의지: 완전 거부 아님
- **시스템 대응**: 협상 가능 영역 제시, 학생 의견 일부 수용
- **성공 가능성**: 협상 성공 시 조건부 수용으로 전환

#### CTX-R-06: 무력감 기반 저항 상황
- **트리거**: "해도 안 된다"는 믿음에서 오는 거부
- **학생 경험**: "어차피 못해요", "해봤자 안 돼요"
- **관찰 지표**:
  - 무력감 표현: 자기효능감 결여
  - 시작 전 포기: 시도 자체 회피
- **시스템 대응**: 극히 작은 성공 경험, 자기효능감 재건
- **주의점**: 장기적 접근 필요, N으로 전환 방지

## N: 무응답형 반응 컨텍스트

### 상황 설명
학생이 개입에 반응하지 않는 상황. 의도적 무시, 관심 부재, 기술적 문제 등 다양한 원인 파악이 중요.

### 학생이 경험하는 세부 UX 컨텍스트

#### CTX-N-01: 완전 침묵 상황
- **트리거**: 메시지 미확인 또는 완전 무시
- **학생 경험**: (응답 없음, 읽지 않음)
- **관찰 지표**:
  - 메시지 상태: 미확인/미읽음
  - 활동 기록: 시스템 접속 없음
- **시스템 대응**: 압박 금지, 다양한 채널 시도, 부모 협조
- **경고 신호**: 7일 이상 지속 시 에스컬레이션

#### CTX-N-02: 최소 반응 상황
- **트리거**: 형식적이고 최소한의 응답만
- **학생 경험**: "네", "아니요", "몰라요"
- **관찰 지표**:
  - 응답 길이: 매우 짧음 (1-3글자)
  - 응답 내용: 무의미한 형식적 답변
- **시스템 대응**: 열린 질문보다 구체적 선택지, 흥미 유발 시도
- **목표**: 응답 길이/깊이 점진적 증가

#### CTX-N-03: 바쁨 핑계 상황
- **트리거**: 시간 부족을 이유로 무응답
- **학생 경험**: "시간 없어요", "바빠요", "나중에요"
- **관찰 지표**:
  - 일관된 핑계: 매번 바쁨 언급
  - 실제 활동: 다른 활동은 함
- **시스템 대응**: 짧은 개입 제안, 시간 협의, 우선순위 재조정
- **확인 필요**: 진짜 바쁨 vs 회피성 바쁨 구분

#### CTX-N-04: 기술적 무응답 상황
- **트리거**: 기술적 문제로 인한 무응답
- **학생 경험**: "알림이 안 왔어요", "앱이 안 돼요"
- **관찰 지표**:
  - 다른 채널 활동: 다른 곳에서는 활동함
  - 간헐적 접속: 가끔 나타남
- **시스템 대응**: 대체 연락 채널 확보, 기술 지원
- **확인 필요**: 기술 핑계 vs 실제 문제 구분

#### CTX-N-05: 선택적 무응답 상황
- **트리거**: 특정 주제/개입에만 무응답
- **학생 경험**: (특정 주제에서만 응답 없음)
- **관찰 지표**:
  - 패턴: 특정 주제 회피
  - 다른 주제 반응: 다른 것에는 응답
- **시스템 대응**: 회피 영역 파악, 우회적 접근, 안전한 환경 조성
- **분석 필요**: 회피 패턴과 원인 파악

#### CTX-N-06: 관망형 무응답 상황
- **트리거**: 지켜보는 중, 아직 반응 안 함
- **학생 경험**: (메시지 확인하지만 응답 안 함)
- **관찰 지표**:
  - 읽음 표시: 확인은 함
  - 응답 없음: 답변은 안 함
- **시스템 대응**: 일관된 메시지 지속, 압박 없이 기다림, 신뢰 구축
- **성공 지표**: 첫 자발적 응답 발생

## D: 지연반응형 반응 컨텍스트

### 상황 설명
학생이 개입에 늦게 반응하는 상황. 숙고형 지연과 회피형 지연 구분이 중요.

### 학생이 경험하는 세부 UX 컨텍스트

#### CTX-D-01: 숙고형 지연 상황
- **트리거**: 신중하게 생각한 후 반응
- **학생 경험**: "생각해볼게요", "좀 이따 말씀드릴게요"
- **관찰 지표**:
  - 지연 이유: 숙고 명시
  - 지연 후 반응: 깊이 있는 답변
- **시스템 대응**: 충분한 시간 제공, 마감 사전 안내, 생각 과정 존중
- **성공 가능성**: 숙고 후 긍정적 결정 비율 높음

#### CTX-D-02: 회피형 지연 상황
- **트리거**: 불편한 상황 회피를 위한 미루기
- **학생 경험**: "내일 할게요", "주말에요", "나중에..."
- **관찰 지표**:
  - 반복적 연기: 계속 미루기
  - 마감 무시: 약속 시간 지키지 않음
- **시스템 대응**: 작은 즉시 행동 유도, 시작 문턱 낮추기, 동행 제안
- **경고 신호**: 무기한 연기, N으로 전환 위험

#### CTX-D-03: 우선순위 조정 지연 상황
- **트리거**: 다른 것이 더 급해서 지연
- **학생 경험**: "시험 끝나고요", "이것부터 끝내고요"
- **관찰 지표**:
  - 구체적 일정: 언제 할지 명시
  - 합리적 이유: 이해 가능한 상황
- **시스템 대응**: 우선순위 존중, 예약/스케줄링 지원, 리마인더 설정
- **성공 가능성**: 계획된 시점에 이행 높음

#### CTX-D-04: 확인 필요 지연 상황
- **트리거**: 더 알고 싶어서 지연
- **학생 경험**: "그게 정확히 뭔데요?", "좀 더 알아보고요"
- **관찰 지표**:
  - 추가 질문: 정보 요청
  - 정보 수집 후 행동: 이해 후 진행
- **시스템 대응**: 상세한 정보 제공, 예시 활용, 질문 환영
- **성공 가능성**: 정보 충족 시 수용 높음

#### CTX-D-05: 기분/컨디션 기반 지연 상황
- **트리거**: 현재 정서 상태로 인한 지연
- **학생 경험**: "오늘 기분이 안 좋아요", "피곤해서요"
- **관찰 지표**:
  - 정서 언급: 기분/컨디션 표현
  - 다른 날 제안: 회복 후 하겠다는 의지
- **시스템 대응**: 정서 상태 존중, 가벼운 활동 제안, 회복 후 리마인더
- **확인 필요**: 일시적 vs 지속적 정서 문제 구분

#### CTX-D-06: 타인 의존 지연 상황
- **트리거**: 다른 사람의 의견/허락 필요
- **학생 경험**: "부모님한테 여쭤볼게요", "친구랑 상의해보고요"
- **관찰 지표**:
  - 타인 언급: 부모/친구/선생님
  - 의존적 결정: 혼자 결정 어려움
- **시스템 대응**: 관련인 개입 지원, 독립적 결정 기회 제공
- **전환 가능성**: 타인 동의 시 A, 반대 시 R로 전환

## 전환 컨텍스트

### 긍정적 전환 컨텍스트

#### CTX-TR-01: R→A 저항 극복 전환
- **트리거**: 저항하던 학생이 수용으로 전환
- **학생 경험**: "그래요, 한번 해볼게요" (이전 저항 후)
- **의미**: 가장 의미 있는 개입 성공
- **시스템 대응**: 전환 인정, 작은 성공 경험 제공, 안정화 지원
- **기록**: 전환 이유 분석, 성공 패턴 저장

#### CTX-TR-02: N→D 반응 유도 전환
- **트리거**: 무응답이던 학생이 지연 반응으로 전환
- **학생 경험**: "생각해볼게요" (이전 무응답 후)
- **의미**: 관계 형성 첫 단계 성공
- **시스템 대응**: 전환 환영, 압박 금지, 신뢰 구축 지속
- **기록**: 전환 계기 분석

#### CTX-TR-03: N→A 직접 수용 전환
- **트리거**: 무응답이던 학생이 직접 수용으로 전환
- **학생 경험**: "네, 할게요" (이전 무응답 후)
- **의미**: 신뢰 형성 완료 신호
- **시스템 대응**: 적극 환영, 성공 경험 축적, 안정화 지원

#### CTX-TR-04: D→A 지연 해소 전환
- **트리거**: 지연하던 학생이 수용으로 전환
- **학생 경험**: "준비됐어요, 시작할게요"
- **의미**: 숙고/준비 완료 신호
- **시스템 대응**: 시작 지원, 초기 성공 경험 제공

### 부정적 전환 컨텍스트

#### CTX-TR-05: A→R 수용에서 저항 전환
- **트리거**: 수용하던 학생이 저항으로 전환
- **학생 경험**: "이거 너무 많아요", "왜 또 해야 해요"
- **의미**: 과부하, 신뢰 손상, 압박 신호
- **시스템 대응**: 즉시 부담 경감, 원인 파악, 관계 회복
- **분석 필요**: 전환 원인 규명

#### CTX-TR-06: A→N 수용에서 무응답 전환
- **트리거**: 수용하던 학생이 무응답으로 전환
- **학생 경험**: (응답 없음)
- **의미**: 관심 상실, 번아웃 신호
- **시스템 대응**: 가벼운 연락 시도, 압박 금지, 정서 확인
- **경고 수준**: 중간 (3회 연속 시 에스컬레이션)

#### CTX-TR-07: R→N 저항에서 무응답 전환
- **트리거**: 저항하던 학생이 완전 무응답으로 전환
- **학생 경험**: (응답 없음)
- **의미**: 완전 단절 위험, 최악의 시나리오
- **시스템 대응**: 즉시 다른 채널 시도, 부모 연락, 다른 에이전트 협조
- **경고 수준**: 높음 (즉시 에스컬레이션)

- **트리거**: 저항에서 무응답으로 전환
- **의미**: 관계 단절 위험
- **시스템 대응**: 모든 압박 중단, 관계 회복 우선, 다중 채널 접근
- **에스컬레이션 대상**: Agent11(정서적 지지), Agent20(학부모 연계)

## 요약: 개입 반응별 핵심 UX 포인트

### 수용(A) 상황 핵심
1. **진정성 확인**: 순응 vs 진정한 수용 구분
2. **지속성 모니터링**: 일회성 아닌 지속적 수용 확인
3. **내면화 지원**: 외재적 → 내재적 동기 전환
4. **과부하 방지**: 열정적 수용 시 번아웃 예방

### 저항(R) 상황 핵심
1. **유형 파악**: 명시적/소극적/협상형 구분
2. **원인 탐색**: 저항의 근본 원인 이해
3. **자율성 존중**: 선택권과 통제감 부여
4. **관계 유지**: 저항 중에도 신뢰 관계 보존

### 무응답(N) 상황 핵심
1. **원인 구분**: 의도적/기술적/선택적 무응답 구분
2. **다양한 접근**: 여러 채널로 시도
3. **인내심**: 압박 없이 일관된 지원
4. **에스컬레이션**: 장기 무응답 시 적절한 개입

### 지연(D) 상황 핵심
1. **유형 구분**: 숙고형/회피형 지연 구분
2. **시간 존중**: 숙고형은 충분한 시간 제공
3. **시작 지원**: 회피형은 첫 걸음 도움
4. **리마인더**: 적절한 상기 시스템 활용
