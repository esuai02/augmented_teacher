## 🔧 주요 개선 사항

### 1. 수학 문제 유형별 침착도 해석 룰 추가 (S12)

**추가된 룰**:
- `R12`: 문제 유형별 침착도 기준선 적용 및 해석
- `R12a`: 계산형 문제 침착도 관리
- `R12b`: 개념형 문제 침착도 관리
- `R12c`: 응용형 문제 침착도 관리
- `R12d`: 증명형 문제 침착도 관리

**DB 연계**: `math_problem_type_calm_db.yaml`

**개선 효과**:
- 계산형: 침착도 85 이하 시 쉬운 계산 문제로 워밍업
- 개념형: 침착도 80 이하 시 기본 개념 복습
- 응용형: 침착도 90 이하 시 쉬운 응용 문제로 단계적 접근
- 증명형: 침착도 75 이하 시 기본 증명 패턴 복습

**현직 선생님 수준 반영**: ✅
- "계산 문제는 침착도 높을 때 정확도 크게 향상"
- "개념 문제는 침착도 낮아도 천천히 생각하면 풀 수 있음"
- "응용 문제는 침착도 높을 때 해결 가능"
- "증명 문제는 침착도 낮을 때 논리 오류 발생"

### 2. 단원별 침착도 관리 룰 추가 (S13)

**추가된 룰**:

- `R13a`: 단원별 침착도 저하 시 단원 특화 개입 전략
- `R13b`: 선행 단원 이해도 부족으로 인한 침착도 저하 감지

**DB 연계**: `math_unit_calm_db.yaml`

**개선 효과**:
- 함수 단원: 침착도 저하 시 기본 그래프 그리기, 개념 복습
- 도형 단원: 침착도 저하 시 기본 도형 문제, 그림 그리기 연습
- 통계 단원: 침착도 비교적 안정적, 기본 통계 문제로 복습
- 선행 단원 이해도 부족 감지 및 복습 권장

**현직 선생님 수준 반영**: ✅
- "함수 단원은 침착도 떨어지기 쉬워, 개념 이해 중요"
- "도형 단원은 공간 지각 능력 필요, 그림 그리기 연습 효과적"
- "통계 단원은 비교적 안정적"
- "선행 단원 이해도 부족은 침착도 저하의 주요 원인"

- `R14d`: 개인 기준선 적용 및 조정

**DB 연계**: `student_level_calm_db.yaml`

**개선 효과**:

- 하위권: 침착도 75 이하 시 기본 개념 재학습, 쉬운 문제 반복

### 4. 수학 오류 유형별 침착도 패턴 분석 룰 추가 (S15)

**추가된 룰**:
- `R15`: 수학 오류 유형별 침착도 패턴 분석
- `R15a`: 계산 실수와 침착도 연관성 분석 (침착도 75 이하에서 주로 발생)
- `R15b`: 개념 오류와 침착도 연관성 분석 (침착도와 무관하게 발생 가능)
- `R15c`: 조건 놓침과 침착도 연관성 분석 (침착도 80 이하에서 주로 발생)
- `R15d`: 수식 오류와 침착도 연관성 분석 (침착도 75 이하에서 주로 발생)

**개선 효과**:
- 계산 실수: 침착도 저하와 밀접한 관련 → 침착도 향상으로 계산 실수 감소
- 개념 오류: 침착도와 상대적으로 독립적 → 개념 이해도 자체 확인 필요
- 조건 놓침: 침착도 저하 시 자주 발생 → 문제 풀기 전 조건 확인 습관 필요
- 수식 오류: 침착도 저하 시 발생 빈도 증가 → 침착도 향상 + 수식 적용 연습

**현직 선생님 수준 반영**: ✅
- "계산 실수는 침착도 낮을 때 자주 나"
- "개념 오류는 침착도 높아도 발생해"
- "조건 놓침은 침착도 낮을 때 주로 발생"
- "수식 오류는 침착도 낮을 때 발생 빈도 증가"

### 5. 문제 난이도별 기준선 차별화 룰 추가 (S16)

**추가된 룰**:
- `R16`: 문제 난이도별 침착도 기준선 차별화
- `R16a`: 기본 문제에서 침착도 저하 시 컨디션 문제 의심
- `R16b`: 심화 문제에서 침착도 저하는 정상 범위로 해석
- `R16c`: 난이도별 차별화된 개입 전략

**개선 효과**:
- 기본 문제: 침착도 저하 → 컨디션 문제 의심 → 휴식 권장
- 심화 문제: 침착도 저하 → 정상 범위 → 단계적 접근 권장
- 난이도별 차별화된 개입 전략 적용

**현직 선생님 수준 반영**: ✅
- "기본 문제에서 침착도 떨어지면 컨디션 문제"
- "심화 문제에서 침착도 떨어지면 정상이야"

### 6. 학습 단계별 침착도 관리 룰 추가 (S17)

**추가된 룰**:
- `R17`: 학습 단계별 침착도 기준 차별화
- `R17a`: 개념 학습 단계에서 침착도 저하는 허용 범위
- `R17b`: 유형 연습 단계에서 침착도 높이기 필요
- `R17c`: 심화 단계에서 침착도 매우 높아야 함

**개선 효과**:
- 개념 학습: 침착도 낮아도 괜찮음 (70~80) → 개념 이해에 집중
- 유형 연습: 침착도 높아야 함 (85 이상) → 침착도 향상 후 연습 진행
- 심화 단계: 침착도 매우 높아야 함 (90 이상) → 침착도 향상 후 심화 학습 진행

**현직 선생님 수준 반영**: ✅
- "개념 학습 단계는 침착도 낮아도 괜찮아"
- "문제 풀이 단계는 높아야 해"
- "심화 단계는 매우 높아야 해"

### 1. 학생 수준 데이터 (student_level_calm_db.yaml)

**데이터 수집 방법**:
- **학생 설문**: "평소 수학 공부할 때 어떤 느낌이 들었나요?"
  - 옵션: 자신 있게 공부할 수 있었어요 / 약간 어려웠지만 공부할 수 있었어요 / 많이 어려웠어요 / 공부하기 전에 두려웠어요
- **선생님 체크리스트**: 
  - "이 학생의 침착도 정상 범위는 얼마인가요?"
  - "이 학생이 침착도가 떨어질 때 주로 어떤 문제가 발생하나요?"
  - "이 학생에게 효과적인 침착도 관리 전략은 무엇인가요?"
- **텍스트 입력**: 선생님의 관찰 기록
  ```
  학생: [학생명]
  수준: [상위권/중위권/하위권]
  관찰 내용: [선생님이 관찰한 침착도 패턴]
  개인 기준선: [개인 평균 침착도]
  개입 전략: [효과적이었던 개입 방법]
  ```

**가정 사항**:
- 학생 수준은 최근 3개월 수학 성적 평균, 학년 평균 대비 백분위, 고난도 문제 해결 능력, 학습 속도 및 이해도를 종합하여 판별
- 개인 기준선은 최근 1개월 침착도 데이터 10개 이상 수집 후 계산
- 학생 수준별 기준선은 학생 설문 및 선생님 체크리스트 데이터를 바탕으로 설정

### 2. 단원별 침착도 특성 데이터 (math_unit_calm_db.yaml)

**데이터 수집 방법**:

  - 옵션: 자신 있게 공부할 수 있었어요 / 약간 어려웠지만 공부할 수 있었어요 / 많이 어려웠어요 / 공부하기 전에 두려웠어요
- **선생님 체크리스트**:
  - "이 단원에서 침착도 관리 포인트는 무엇인가요?"
  - "이 단원에서 주로 발생하는 침착도 저하 원인은 무엇인가요?"
  - "이 단원에서 효과적인 침착도 관리 전략은 무엇인가요?"
- **텍스트 입력**: 선생님의 관찰 기록
  ```
  단원: [단원명]
  관찰 내용: [선생님이 관찰한 침착도 패턴]
  개입 전략: [효과적이었던 개입 방법]
  효과: [개입 후 결과]
  ```

**가정 사항**:
- 단원별 침착도 기준선은 해당 단원 학습 중 수집된 침착도 데이터의 평균값 사용
- 단원별 특성(침착도 안정성, 저하 경향)은 학생 설문 및 선생님 체크리스트 데이터를 바탕으로 설정
- 선행 단원 이해도는 해당 단원 관련 문제의 정답률로 측정 (임계값: 70%)
