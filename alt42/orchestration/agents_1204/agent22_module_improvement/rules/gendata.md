## 🗺️ 온톨로지 매핑 (alphatutor_ontology.owl)

위의 100개 데이터 항목은 Agent 22의 모듈 성능 개선 및 자가 업그레이드 기능을 위해 필요합니다. 주요 데이터 항목들은 다음과 같이 온톨로지 클래스로 매핑됩니다:

### 실행 및 성능 관련 데이터
- 워크플로우 실행 ID, 실행 타임스탬프, 실행 지속시간 → `AgentExecutionLog`
- 에이전트 버전, rules.yaml 버전 태그 → `ModuleImprovement`
- 룰별 실행 횟수, 룰별 평균 실행시간, 룰별 실패 수치 → `RuleActivationFrequency`
- 컨테이너/VM 메모리 사용량, CPU 사용률, 디스크 I/O 지표 → `ResourceUsage`

### 실패 및 오류 관련 데이터
- 룰별 예외 스택트레이스, 에러 코드, 에러 심각도 → `FailurePattern`
- 재시도 횟수, 타임아웃 발생 여부 → `ErrorSelfRecovery`
- 미처리 케이스 목록, 미처리 케이스 발생 빈도 → `EdgeCaseMissing`

### 룰 네트워크 관련 데이터
- 룰 충돌 탐지 플래그, 룰 비활성화/스킵된 항목 목록 → `RuleConflictPossibility`
- 룰 의존성 맵, 룰 커버리지 측정값 → `RuleNetworkOptimization`
- 규칙 수정 제안, 제안 수락/거부 이력 → `SelfUpgradeIdea`

### 자동 진화 관련 데이터
- 자동교정 시도 기록, 자동교정 실패 원인 → `AutoEvolutionStructure`
- 데이터 수집, 취약점 진단 → `VulnerabilityAnalysis`
- 3-File 생성 (rules.yaml, metadata.md, dataindex.html) → `ThreeFileSystemDocument`
- 검증, 배포, 롤백 조건 → `Verification`, `Deployment`

### 시스템 안정성 관련 데이터
- 성능 변화 트렌드, 성능 회귀 탐지 → `SystemInefficiencyInstability`
- 서비스 레벨 지표(SLI), 서비스 레벨 목표(SLO) → `LongTermStability`
- 모니터링 대시보드, 알림 발송 기록 → `VerificationSystem`

자세한 온톨로지 매핑 정보는 `metadata.md` 파일을 참조하세요.
